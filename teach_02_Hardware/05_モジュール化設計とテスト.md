# Raspberry Pi PicoとMicroPythonによるモジュール化設計

プログラムの規模が大きくなるにつれ、コードを整理して品質を保つ手法が重要になります。本教材では、関数やクラスを使ったプログラムの分割、モジュール化による開発効率向上、段階的なテスト（ユニットテストと統合テスト）、およびハードウェアとソフトウェアの責任分離による障害追跡といった概念を学びます。

例としてLED点滅、UART通信、PWM制御、タクトスイッチ入力を扱い、コード例は基本的にMicroPython（必要に応じてC/C++の補足）で示します。各セクションでは概念の解説とそれに続く段階的な演習を含み、簡単な模擬バグの事例とその解決法も紹介します。モジュール化設計と段階的テストが組み込み開発にもたらす利点を体感していきましょう。

## 1. 関数によるプログラムの分割
プログラムを整理する第一歩として「`関数（function）`」の活用があります。
MicroPythonにおいて、関数は特定のタスクを実行するステートメントの集まりです。関数を使うことで、プログラムをより小さなモジュール化されたブロックに分割できます。
コードの規模が大きくなるにつれて、関数で処理を分けることでコードが整理され、管理しやすくなります。また、同じ処理を繰り返し書く必要がなくなり重複を避け、一度書いたコードを再利用できるというメリットもあります。

### 関数の基本と利点

関数は`def`キーワードで定義し、名前と必要に応じて引数を指定します。以下に簡単な関数定義の例を示します。

```py
def greet(name):
    print(name + "さん、こんにちは！")
```

上記の`greet`関数は名前を受け取り、挨拶のメッセージを表示します。関数を定義することで、この挨拶処理をプログラム中どこからでも簡単に呼び出すことができます。関数を使う主な利点は次の通りです。

* **コードの整理**：関連する処理を一つのまとまり（関数）として定義し、プログラム全体を見通しやすくします。大きなプログラムでも、小さな部品に分割されていれば理解しやすくなります。
* **再利用性**：一度定義した関数は、他の場所から何度でも呼び出せます。同じコードをコピー＆ペーストする必要がなくなり、修正も一箇所で済みます。
* **保守性**：バグ修正や機能追加の際、関数ごとに処理が分かれていれば影響範囲を限定しやすく、変更やテストが容易です。

## 1.2 LED点滅プログラムで見る関数の活用

それでは具体例として、LEDを点滅させるプログラムで関数の使い方を見てみましょう。Raspberry Pi Picoのボード上には組み込みLED（Pico無印ではGPIO25に接続されたLED）があり、これを点滅（いわゆる「Lチカ」）させることが入門としてよく行われます。 まずは関数を使わずにLEDを点滅させるコードを書いてみます。
```py

import machine
import time

led = machine.Pin(25, machine.Pin.OUT)  # Picoの組み込みLED (GPIO25)
for i in range(10):
    led.value(1)       # LED ON
    time.sleep(0.5)   # 0.5秒間待つ
    led.value(0)       # LED OFF
    time.sleep(0.5)   # 0.5秒間待つ
```
このコードでは、LEDを10回点滅させています。しかし、LEDを点滅させる処理（ON->待機->OFF->待機）が直接プログラム本体に書かれており、もし他の箇所でも同様の点滅処理をしたい場合はコードの重複が発生します。また、点滅の間隔を変更したい場合にも、直接数値を書き換える必要があります。ここで関数を使うことで改善できます。

次に、LED点滅処理を**関数化**してみましょう。


```py
import machine, time

led = machine.Pin(25, machine.Pin.OUT)

def blink_led(pin_obj, times, interval):
    """指定したピンのLEDをtimes回点滅させる関数"""
    for i in range(times):
        pin_obj.value(1)          # LED ON
        time.sleep(interval)     # interval秒待つ
        pin_obj.value(0)          # LED OFF
        time.sleep(interval)     # interval秒待つ

blink_led(led, 10, 0.5)  # LEDを0.5秒間隔で10回点滅
```

関数`blink_led(pin_obj, times, interval)`を定義し、LED点滅のロジックをその中にまとめました。これにより、メイン部分では`blink_led`関数を呼ぶだけで目的の処理が行われます。関数化の利点は以下のように現れています。

* メインのコードが簡潔になり、処理の意図が明確になる（ 「LEDを10回点滅する」と一行で表現できるようになる）。
* 点滅の処理を変更したい場合（例えば点滅回数や間隔）、関数内のコードか呼び出しパラメータを変更すればよく、重複している複数箇所を修正する必要がない。
* 別のピンに接続されたLEDを点滅させる場合でも、`machine.Pin`のオブジェクトを変えて`blink_led`を呼ぶだけで再利用できる。

※「関数を使う」ことを「関数を呼ぶ(call)」と言ったりします

### 演習1:
上記のblink_led関数を応用して、任意の回数・任意の間隔でLEDを点滅させるプログラムを書いてみましょう。  
例えば、「1秒間隔で5回点滅した後、0.2秒間隔で5回点滅する」ようなシーケンスを関数呼び出しの組み合わせで実現できます。実行して結果（LEDの点滅パターン）を観察し、期待通りになっているか確認してみてください。
